COMMIT: MatrixSwarm – Vault Creation Restoration & Security Patch

Date: 2026-02-03
Scope: Phoenix → Registry Manager
Requires: Phoenix reload after update.

Overview

This update fixes a registry crash that occurred when adding new registry objects through the Registry Manager dialog. The issue was caused by a debug print statement referencing a specific namespace key (discord) inside the registry store.
The debug call attempted to access:

Fix Summary

The debug call attempted to access:
```bash
  self.registry_store.get_data()["discord"]
``` 

### Fix Summary

 - registry_manager.py

   - Removed the debug statement:

```bash
print("Store registry obj id:", id(self.registry_store.get_data()["discord"]))
```

 - This statement assumed the discord namespace always existed in the registry store.

 - Registry namespaces are created dynamically, so referencing a fixed key caused crashes during object creation.

 - The remaining debug line now safely logs the namespace instance instead:

```bash 
print("Namespace object id:", id(ns))
```
### Result

 - Registry objects can now be added without crashing the GUI.

 - Namespace creation works correctly even when a class has not yet been initialized in the registry.

 - Registry operations remain stable across all connector classes.
